# AnyBot 技术方案 — 手机控制 Mac + AI Agent 自动化

## 一、项目定位

不仅是一个「手机远程控制电脑」的工具，而是一个**通用的计算机操控平台**：
- **Phase 1**: 人通过手机浏览器操控 Mac ✅ **P0 已完成**
- **Phase 2**: AI Agent 通过相同接口操控 Mac（对标 Anthropic Computer Use / OpenAI CUA）
- **Phase 3**: 人 + AI 混合操控，人在手机上监督 AI 的自动化操作

因此架构设计的核心原则：**屏幕观察和操控能力必须抽象为统一接口，人和 AI 共用。**

## 三、为什么选 Python 而不是 Swift

| 维度 | Python | Swift (原生) |
|------|--------|-------------|
| AI/LLM 生态 | ✅ 最强，Anthropic/OpenAI SDK 都是 Python 优先 | ❌ 几乎没有 |
| 自动化生态 | ✅ pyautogui, pyobjc, selenium, playwright | ⚠️ 有但社区小 |
| 开发速度 | ✅ 快，适合快速迭代 | ❌ 编译型语言，迭代慢 |
| 屏幕捕获性能 | ⚠️ mss 库足够(30fps+) | ✅ ScreenCaptureKit 最优 |
| 跨平台潜力 | ✅ 未来扩展 Windows/Linux 零成本 | ❌ 仅 macOS |
| Computer Use 参考实现 | ✅ Anthropic 官方 demo 就是 Python | - |

**结论：选 Python。** 屏幕捕获性能用 mss（底层调用 Quartz）已经够用，关键是 AI Agent 集成和跨平台扩展。

## 四、核心模块技术选型

### 4.1 屏幕捕获模块

```python
# 方案：mss (多平台截屏库，底层调用 macOS Quartz API)
# PyPI: 67M+ 下载, 活跃维护
import mss
import mss.tools

with mss.mss() as sct:
    # 捕获主屏幕
    monitor = sct.monitors[1]  # 主屏幕
    img = sct.grab(monitor)    # 返回 BGRA 像素数据
    # 转为 JPEG/PNG 用于传输或 AI 分析
```

**为什么选 mss：**
- 纯 Python，无需编译，pip install 即可
- macOS 底层用 Quartz (`CGWindowListCreateImage`)，性能好
- 30fps+ 截屏无压力（1080p 单帧 <20ms）
- Anthropic 的 `mac_computer_use` 项目也是类似方案
- 跨平台：Windows 用 DXGI，Linux 用 Xlib

**备选：**
- `Pillow.ImageGrab` — 更慢，但更通用
- `pyobjc + Quartz` — 直接调用 macOS API，最底层但最灵活

### 4.2 输入控制模块

```python
# 方案：pyautogui (鼠标/键盘模拟, 跨平台)
# PyPI: 200M+ 下载
import pyautogui

# 鼠标操作
pyautogui.moveTo(x, y)           # 移动到坐标
pyautogui.click(x, y)            # 左键点击
pyautogui.doubleClick(x, y)      # 双击
pyautogui.rightClick(x, y)       # 右键
pyautogui.scroll(clicks)         # 滚轮
pyautogui.moveTo(x, y, duration=0.5)  # 带动画移动

# 键盘操作
pyautogui.typewrite('hello')     # 输入文本
pyautogui.hotkey('command', 'c') # 组合键 (Cmd+C)
pyautogui.press('enter')         # 单键
```

**为什么选 pyautogui：**
- Anthropic Computer Use / OpenAI CUA 的 Mac 适配都用它
- macOS 底层调用 Quartz Event (CGEvent)，和直接写 Swift 效果一样
- 跨平台支持
- API 简单直观，和 AI Agent 的 action 映射非常直接

**补充：`cliclick` (Homebrew)**
- 某些 macOS 场景 pyautogui 权限受限时的备选
- `deedy/mac_computer_use` 用的就是 cliclick
- `brew install cliclick && cliclick c:100,200` (点击坐标 100,200)

### 4.3 画面传输模块

#### 阶段一（P0）: WebSocket + MJPEG — 快速验证

```
Mac端: mss截屏 → JPEG压缩 → WebSocket推送
手机端: WebSocket接收 → Canvas/img绘制
```

**技术栈：**
- Mac 端：`FastAPI` + `websockets` (或 `python-socketio`)
- 手机端：纯 HTML5 + JavaScript
- 帧率：15-30 fps，局域网延迟 50-100ms
- 带宽：720p JPEG quality=50 约 30-50KB/帧，30fps ≈ 1-1.5MB/s

```python
# Mac 端核心代码示意
from fastapi import FastAPI, WebSocket
import mss, io
from PIL import Image

app = FastAPI()

@app.websocket("/ws/screen")
async def screen_stream(websocket: WebSocket):
    await websocket.accept()
    with mss.mss() as sct:
        monitor = sct.monitors[1]
        while True:
            img = sct.grab(monitor)
            # 转 JPEG 压缩
            pil_img = Image.frombytes("RGB", img.size, img.bgra, "raw", "BGRX")
            pil_img = pil_img.resize((1280, 800))  # 缩放降低带宽
            buf = io.BytesIO()
            pil_img.save(buf, format='JPEG', quality=50)
            await websocket.send_bytes(buf.getvalue())
            await asyncio.sleep(1/30)  # 30fps
```

#### 阶段二（P1）: WebRTC — 生产级低延迟

```
Mac端: mss截屏 → H.264编码 → WebRTC PeerConnection
手机端: 浏览器原生 RTCPeerConnection 接收 → <video> 播放
控制指令: RTCDataChannel 双向传输
```

**技术栈：**
- Mac 端：`aiortc` (Python WebRTC 库, GitHub 4k+ stars, 活跃维护)
- 信令：FastAPI WebSocket (仅用于 SDP/ICE 交换)
- 编码：aiortc 内置 VP8/H.264 编码
- 延迟：局域网 20-50ms

## 五、统一操控 API（核心设计，面向 AI Agent）

这是整个项目的**关键抽象层**。人和 AI 都通过这套 API 操控电脑。

### 5.1 API 设计（对标 Anthropic Computer Use + OpenAI CUA）

```python
# === 操控动作定义 ===

class Action:
    """所有操控动作的基类"""
    pass

class ScreenshotAction(Action):
    """截取当前屏幕"""
    # 返回: base64 编码的 PNG/JPEG 图片
    pass

class ClickAction(Action):
    """鼠标点击"""
    x: int           # 屏幕坐标 x
    y: int           # 屏幕坐标 y
    button: str      # "left" | "right" | "middle"
    click_type: str  # "single" | "double" | "triple"

class TypeAction(Action):
    """键盘输入"""
    text: str        # 要输入的文本

class KeyAction(Action):
    """按键/组合键"""
    keys: list[str]  # 如 ["command", "c"] 表示 Cmd+C

class ScrollAction(Action):
    """滚轮"""
    x: int
    y: int
    direction: str   # "up" | "down" | "left" | "right"
    amount: int      # 滚动量

class DragAction(Action):
    """拖拽"""
    start_x: int
    start_y: int
    end_x: int
    end_y: int

class CursorPositionAction(Action):
    """获取当前光标位置"""
    pass

class WaitAction(Action):
    """等待指定时间"""
    duration: float  # 秒
```

### 5.2 API 端点

```
# REST API
GET  /api/screenshot      — 获取当前屏幕截图 (JPEG)
GET  /api/screenshot/base64 — 获取 base64 编码截图（供 AI Agent 使用）
GET  /api/screen/info     — 获取屏幕分辨率等信息
GET  /api/cursor          — 获取当前光标位置
GET  /api/windows         — 获取窗口列表（置顶窗口优先）

POST /api/window/pin      — 置顶窗口
POST /api/window/unpin    — 取消置顶窗口
POST /api/action          — 执行单个操控动作
POST /api/actions         — 批量执行动作序列
GET  /health              — 健康检查

# WebRTC 信令
POST /api/webrtc/offer    — WebRTC SDP Offer/Answer 交换

# WebSocket
WS   /ws/screen           — 实时画面流 (MJPEG 回退模式)
WS   /ws/control          — 实时操控通道 (MJPEG 回退模式)

# WebRTC 数据通道
RTCDataChannel            — 实时操控通道 (WebRTC 模式，延迟更低)
```

### 5.3 和 AI Agent 的集成方式

```python
# === Anthropic Computer Use 集成示例 ===

import anthropic

client = anthropic.Anthropic()

# Computer Use Tool 定义（Anthropic 标准格式）
computer_tool = {
    "type": "computer_20241022",  # Anthropic computer use tool
    "name": "computer",
    "display_width_px": 1280,
    "display_height_px": 800,
    "display_number": 1,
}

# AI Agent 循环
def agent_loop(task: str):
    messages = [{"role": "user", "content": task}]
    
    while True:
        response = client.messages.create(
            model="claude-sonnet-4-20250514",
            max_tokens=4096,
            tools=[computer_tool],
            messages=messages,
        )
        
        # 处理 AI 返回的 tool_use
        for block in response.content:
            if block.type == "tool_use":
                action = block.input  
                # action 格式如: {"action": "click", "coordinate": [100, 200]}
                # 或: {"action": "screenshot"}
                # 或: {"action": "type", "text": "hello"}
                
                result = execute_action(action)  # 调用我们的统一操控 API
                
                messages.append({"role": "assistant", "content": response.content})
                messages.append({
                    "role": "user", 
                    "content": [{"type": "tool_result", "tool_use_id": block.id, "content": result}]
                })
        
        if response.stop_reason == "end_turn":
            break

# execute_action 内部就是调用我们的 /api/action 端点
# 无论是人在手机上点击，还是 AI 发出的指令，最终都走同一条路径
```

## 十、对标分析：Anthropic Computer Use 的工作原理

这是我们 AI Agent 集成时要对接的核心协议：

```
Anthropic Computer Use 交互循环：

1. 用户给任务 → "帮我在 Chrome 里搜索天气"
2. Claude 返回 tool_use: {"action": "screenshot"} 
3. 我们截屏 → base64 图片返回给 Claude
4. Claude 分析截图，返回 tool_use: {"action": "click", "coordinate": [500, 300]}
5. 我们执行点击 → 截屏 → 返回给 Claude
6. Claude 继续分析... 直到任务完成

关键点：
- Claude 通过截图"看到"屏幕（不是 DOM/Accessibility Tree）
- Claude 输出的是坐标级别的操控指令
- 整个循环就是 截图→思考→操作→截图→思考→操作...
- 这和我们的统一操控 API 完美匹配
```

**这意味着：**
- 我们的 `GET /api/screenshot` 直接给 AI 用
- 我们的 `POST /api/action` 直接执行 AI 的指令
- 手机端只是这个循环的"可视化监控面板"
- 人和 AI 的操控在底层完全统一

## 十一、关键设计决策总结

| 决策 | 选择 | 理由 |
|------|------|------|
| 主语言 | **Python** | AI/LLM 生态第一，Anthropic/OpenAI 官方都用 Python 做 Computer Use |
| 屏幕捕获 | **mss** | 67M+ 下载，跨平台，macOS 底层 Quartz，30fps+ |
| 输入控制 | **pyautogui + pyobjc** | pyautogui 跨平台基础操控，pyobjc AXUIElement 精确窗口激活 |
| Web 框架 | **FastAPI** | 异步高性能，WebSocket 原生支持，适合实时应用 |
| 画面传输 | **WebRTC (主) + MJPEG (回退)** | WebRTC H.264 低延迟低带宽，MJPEG 作为兼容性回退 |
| 控制通道 | **RTCDataChannel (主) + WebSocket (回退)** | DataChannel 延迟更低，drag_move 走 fast-track |
| API 设计 | **对标 Anthropic/OpenAI 协议** | 未来 Agent 集成零适配成本 |
| 手机端 | **纯 HTML5 + Vanilla JS** | 无依赖，浏览器直接打开，兼容性最好 |
| 认证 | **Token (SHA-256)** | 支持密码/环境变量/文件三种方式，REST Bearer + WS 查询参数 |
| AI Agent | **Anthropic Computer Use (computer_20250124)** | 成熟的屏幕操控协议，截图→分析→操控循环 |
| MCP Server | **FastMCP (stdio 模式)** | 独立进程，HTTP 代理到本地 REST API，适配 Claude Desktop / Cursor |
